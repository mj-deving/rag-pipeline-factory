## What We're Building

A **meta-workflow** for n8n that builds other RAG workflows. The user describes what they want (data sources, vector store, query interface) via a chat trigger or webhook, and the factory:

1. Parses the request into a structured spec
2. Researches available n8n nodes via `n8nac skills search`
3. Generates a **document ingestion workflow** (.workflow.ts) — loading, chunking, embedding, vector storage
4. Generates a **query/retrieval workflow** (.workflow.ts) — retrieval, optional reranking, response generation
5. Pushes both to n8n via `n8nac push`
6. Validates via `n8nac verify`
7. Tests the ingestion pipeline with a sample document
8. Reports results with workflow IDs and test outcomes

This is **code writing code** — the ultimate code-mode showcase. The AI agent writes TypeScript that generates TypeScript workflow definitions.

## Why This Matters

- **613 RAG templates** exist in the n8n community, but no factory to generate them from a description

## Architecture Design

### Option A: Single Factory Workflow (Recommended for v0.1)

One n8n workflow with a Chat Trigger that accepts natural language requests:

```
Chat Trigger → AI Agent (Haiku) → Code-Mode Tool (factory logic) → Response
```

The Code-Mode tool contains the factory logic:
- Parse the user's request into a structured spec
- Use `this.helpers.httpRequest()` to call n8n API directly for node search/push/verify
- Generate .workflow.ts content as strings
- Push generated workflows via n8n API
- Return results to the chat

### Option B: Multi-Workflow Architecture (v1.0)

```
Factory Chat UI (Chat Trigger)
  → Spec Generator Agent (parses request into structured spec)
  → Pipeline Builder Agent (generates ingestion workflow)
  → Query Builder Agent (generates retrieval workflow)
  → Deployer Agent (pushes, verifies, tests)
  → Reporter (summarizes results)
```

**Start with Option A.** It's simpler, proves the concept, and can evolve to Option B.
